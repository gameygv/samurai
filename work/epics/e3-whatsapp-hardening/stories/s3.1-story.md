# S3.1: Renovar token Meta Cloud API

## User Story

Como operador del CRM, quiero que el bot pueda enviar mensajes por WhatsApp,
lo cual requiere un token válido de Meta Cloud API.

## Context

El token actual expiró el 2026-03-30 17:00 PDT. Error exacto en activity_logs:

```
ERROR WHATSAPP: "Error validating access token: Session has expired on
Monday, 30-Mar-26 17:00:00 PDT. The current time is Monday, 30-Mar-26
18:01:21 PDT." (OAuthException, code 190, error_subcode 463)
```

El token se almacena en `whatsapp_channels.api_key` para el canal Meta
(id: `40c90fef-900c-47ef-8624-8aeed83f8e31`, name: "Whatsapp API Test",
instance_id: `863928923479101`).

### Cómo renovar
1. Ir a Meta Developer Dashboard → tu app → WhatsApp → Configuration
2. Generar un nuevo System User token con permisos: `whatsapp_business_messaging`, `whatsapp_business_management`
3. Preferir "Permanent Token" (System User) sobre "Temporary Token" (60 días)
4. Actualizar `api_key` en `whatsapp_channels` para el canal Meta

### Cómo verificar
- Enviar un mensaje de prueba desde el Panel del CRM
- Verificar en activity_logs que NO aparezca "OAuthException"
- Verificar que el mensaje IA aparezca en conversaciones con `message_id` (wamid)

## Acceptance Criteria

- [ ] Token renovado en whatsapp_channels.api_key
- [ ] Mensaje de prueba enviado exitosamente desde el CRM
- [ ] Respuesta IA guardada en conversaciones con wamid
- [ ] Sin errores OAuthException en activity_logs

## Notes

Esta es una tarea manual (actualizar token en BD), no requiere cambio de código.
Pero es P0 porque sin token el bot está completamente inoperativo.
