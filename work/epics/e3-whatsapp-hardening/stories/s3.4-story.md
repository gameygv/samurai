# S3.4: Eliminar/deshabilitar provider GOWA

## User Story

Como operador del CRM, no quiero que mensajes se envíen accidentalmente por
un provider no oficial (GOWA) que está deprecated y con servicio inestable.

## Context

GOWA (Evolution API) era el provider original de WhatsApp antes de migrar
a la API oficial de Meta. Hallazgos de E2:

- Canal "Gowa 01" sigue activo (id: `a488f580-20bc-460e-98e7-56d9741d3c4e`)
- `send-message-v3` tiene bloques completos de código para GOWA (líneas 71-90)
- Ya existe feature flag (commit `d2f62ca`) para ocultar providers no oficiales en UI
- El server GOWA (`gowa.poesis.net`) mostró timeout durante E2E testing
- E2 cambió el default provider de 'gowa' a 'meta', pero el código GOWA sigue ahí

### Blast radius de send-message-v3 (del AR de E2)

| Caller | Puede caer en GOWA? |
|--------|---------------------|
| process-samurai-response | Solo si lead.channel_id apunta a canal GOWA |
| messagingService.ts (Panel) | Solo si el canal explícito es GOWA |
| process-campaign-queue | Solo si lead tiene canal GOWA |
| process-credit-reminders (agente) | Sí — sin lead_id, fallback chain puede llegar a GOWA |

### Opciones de implementación
1. **Soft delete:** Desactivar canal GOWA (`is_active = false`) + remover código GOWA de send-message-v3
2. **Hard delete:** Eliminar canal de BD + código + UI references
3. **Solo desactivar:** `is_active = false` sin tocar código

### Leads con canal GOWA
Antes de eliminar, verificar cuántos leads tienen `channel_id` apuntando
al canal GOWA. Esos leads necesitarían reasignarse al canal Meta.

```sql
SELECT COUNT(*) FROM leads WHERE channel_id = 'a488f580-20bc-460e-98e7-56d9741d3c4e';
```

### Archivos a modificar (opción soft delete)
- `supabase/functions/send-message-v3/index.ts` — remover bloque GOWA (líneas 71-90)
- `supabase/functions/evolution-webhook/index.ts` — remover parsing Evolution API (líneas 81-88)
- `supabase/functions/setup-whatsapp-webhook/index.ts` — remover setup GOWA
- `src/components/settings/ChannelsTab.tsx` — ya tiene feature flag, confirmar que oculta GOWA
- BD: desactivar canal, reasignar leads

## Acceptance Criteria

- [ ] Código GOWA removido de send-message-v3 (o marcado deprecated)

- [ ] Ningún caller puede caer en GOWA por fallback
- [ ] credit-reminders (agente) usa canal default Meta, no fallback
