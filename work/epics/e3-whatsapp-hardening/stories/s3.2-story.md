# S3.2: Limpiar datos de prueba E2E

## User Story

Como operador del CRM, no quiero ver leads ficticios de pruebas automatizadas
mezclados con leads reales en mi pipeline de ventas.

## Context

Durante el E2E testing de E2 (S2.2) se crearon 3 leads de prueba en producción:

| Lead Name | Phone | Created At | Lead ID |
|-----------|-------|------------|---------|
| E2E Test Lead | 5215500000001 | ~02:34 UTC | f6d27478-88fa-44e3-a6ae-0de15cfe98bc |
| E2E Test V2 | 5215500000002 | ~02:45 UTC | 4eb745fc-7f3c-402c-a791-499d37615f31 |
| E2E Test V3 | 5215500000003 | ~02:49 UTC | e7957608-bd13-435a-8cc8-8f1d957b9a77 |

Cada uno tiene conversaciones asociadas y entries en activity_logs.

### Limpieza necesaria
```sql
-- Conversaciones se borran por CASCADE (FK con ON DELETE CASCADE)
DELETE FROM leads WHERE telefono IN ('5215500000001', '5215500000002', '5215500000003');
-- Activity logs no tienen FK a leads, se pueden dejar (audit trail)
```

## Acceptance Criteria

- [ ] Los 3 leads de prueba eliminados de producción
- [ ] Conversaciones asociadas eliminadas (CASCADE)
- [ ] Verificar que no hay otros leads de prueba residuales
