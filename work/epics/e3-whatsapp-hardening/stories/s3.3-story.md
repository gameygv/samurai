# S3.3: Delivery checkmarks en ChatViewer

## User Story

Como operador del CRM, quiero ver el estado de entrega de cada mensaje
enviado por el bot (enviado/entregado/leído), similar a los checkmarks
de WhatsApp, para saber si el cliente recibió la respuesta.

## Context

E2 (S2.3) ya implementó todo el backend:
- `evolution-webhook` procesa status webhooks y actualiza `delivery_status`
- Valores posibles: `null`, `sent`, `delivered`, `read`, `failed`
- `useRealtimeMessages.ts` hace `select('*')`, así que `delivery_status` ya llega al frontend

Solo falta el componente visual.

### Archivos relevantes
- `src/hooks/useRealtimeMessages.ts` — ya trae delivery_status (select *)
- `src/components/ChatViewer.tsx` — renderiza mensajes, aquí va el cambio principal
- Posible: `src/components/chat/MessageList.tsx` o similar si el render de mensajes está separado

### Diseño visual propuesto (estilo WhatsApp)
- `null` (Panel/sin tracking) → sin icono
- `sent` → ✓ gris
- `delivered` → ✓✓ gris
- `read` → ✓✓ azul
- `failed` → ✗ rojo + tooltip con "Error de entrega"

Solo mostrar en mensajes con `emisor` = 'IA' o 'SAMURAI' o 'BOT' (mensajes salientes).
No mostrar en mensajes de CLIENTE (entrantes) ni HUMANO desde Panel (no tienen tracking de Meta).

### Polling
El hook ya hace polling cada 10s, así que los checkmarks se actualizarán
automáticamente cuando Meta envíe el status webhook. No es "tiempo real"
estricto pero es suficiente.

## Acceptance Criteria

- [ ] Mensajes salientes (IA/BOT/SAMURAI) muestran checkmark de delivery
- [ ] Checkmarks reflejan: sent (✓), delivered (✓✓), read (✓✓ azul), failed (✗ rojo)
- [ ] Mensajes sin delivery_status no muestran nada
- [ ] Mensajes entrantes (CLIENTE) no muestran checkmarks
- [ ] Estilo coherente con el tema oscuro del CRM
