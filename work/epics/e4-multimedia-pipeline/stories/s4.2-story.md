# S4.2: Media Tag Parser in AI Response

## User Story

As a **WhatsApp end-user**,
I want to **receive images inline when the AI includes media in its response**,
so that **I see product photos and visual content directly in the chat**.

## Acceptance Criteria

### Scenario: AI response contains a media tag
```gherkin
Given the AI produces a response containing <<MEDIA:https://example.com/image.jpg>>
When process-samurai-response processes the AI text
Then the media tag is extracted and the URL is parsed
And the remaining clean text has the tag removed
```

### Scenario: Clean text is 1024 chars or fewer (caption mode)
```gherkin
Given the clean text after tag removal is <= 1024 characters
When the message is sent via send-message-v3
Then the image is sent with mediaData containing the URL
And the clean text is included as caption on the image message
```

### Scenario: Clean text exceeds 1024 chars (split mode)
```gherkin
Given the clean text after tag removal is > 1024 characters
When the message is sent via send-message-v3
Then the image is sent with mediaData (no caption)
And a separate text message is sent with the clean text
```

### Scenario: No media tag in AI response
```gherkin
Given the AI response does not contain any <<MEDIA:...>> tag
When process-samurai-response processes the AI text
Then the current message flow is used unchanged
```
