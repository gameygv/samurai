# S4.3 Design: Edge Cases & Observability

## Problem

S4.1 and S4.2 handle happy path but lack defensive checks for edge cases
(oversized audio, invalid media URLs) and success logging for observability.

## Changes

### 1. Audio size check in transcribe-audio (before Whisper call)
- After downloading audio blob, check blob.size > 25MB (Whisper limit)
- If too large: logAndFallback with descriptive message
- Location: transcribe-audio/index.ts, after line 60 (audioBlob)

### 2. Media URL validation in process-samurai-response (before send)
- After extracting mediaUrl from regex, validate it starts with https://
- If invalid: log warning, skip media send, send cleanText as plain text
- Location: process-samurai-response/index.ts, after line 138 (mediaMatch block)

### 3. Success logging for media sends
- After successful media send in process-samurai-response, log to activity_logs
- Location: process-samurai-response/index.ts, after wamid assignment in media branches
