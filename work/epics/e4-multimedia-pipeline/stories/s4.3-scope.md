# S4.3 Scope: Edge Cases & Observability

## In Scope

- Audio >25MB detection and graceful fallback in `transcribe-audio`
- activity_log entry for oversized audio files
- Basic URL validation for media URLs in AI response (https, valid format)
- Fallback to text-only when media URL is invalid in `process-samurai-response`
- activity_log entry for invalid media URLs
- activity_log entry for successful image sends (lead_id, media_url, mode)
- activity_log entry for successful transcriptions (already exists from S4.1, verify)

## Out of Scope

- HEAD request or reachability check on media URLs (basic format check only)
- Retry logic for failed media sends
- Retry logic for failed transcriptions
- Dashboard or UI for activity_logs
- Alerting on failure thresholds
- Audio format conversion or compression
- Multiple media tags per response

## Where

- `supabase/functions/transcribe-audio/index.ts` — audio size check, transcription logging
- `supabase/functions/process-samurai-response/index.ts` — URL validation, image send logging

## Decisions Referenced

- D3: Fallback de Whisper — respuesta generica, bot pide que repita + log
- D5: Caption en imagen, fallback a separados si >1024 chars

## Done Criteria

- [ ] Audio >25MB triggers fallback with activity_log entry
- [ ] Invalid media URL in AI response falls back to text-only with activity_log entry
- [ ] Successful image sends logged in activity_logs
- [ ] Successful transcriptions logged in activity_logs
- [ ] Valid media URL check (basic format) before attempting send
- [ ] Zero regression on existing text, audio, and image flows
