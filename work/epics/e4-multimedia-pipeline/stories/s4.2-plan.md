## T1: Add media tag parser and routing logic

**File:** `supabase/functions/process-samurai-response/index.ts`
**Location:** After line 125 (`const aiText = ...`), before the `if (aiText)` block.

**Steps:**
1. After `aiText` assignment (line 125), add regex extraction:
   - Pattern: `<<MEDIA:(https?:\/\/[^>]+)>>`
   - Extract first match as `mediaUrl`
   - Generate `cleanText` by removing the tag and trimming
2. Inside the `if (aiText)` block, replace the direct send-message-v3 call with branching logic:
   - **Has media tag + cleanText <= 1024 chars (caption mode):**
     Call send-message-v3 with `mediaData: { url: mediaUrl, type: 'image' }` and `message: cleanText`
   - **Has media tag + cleanText > 1024 chars (split mode):**
     Call send-message-v3 with `mediaData: { url: mediaUrl, type: 'image' }` and `message: ''` (image only),
     then call send-message-v3 again with `message: cleanText` (text only)
   - **No media tag:** existing flow unchanged
3. Conversation insert uses `cleanText || aiText` as the stored message (keep full text in DB with tag stripped)
4. Deploy: `supabase functions deploy process-samurai-response`

**Verification:** Function deploys without errors.

**Commit:** `feat(s4.2): T1 media tag parser with caption/split routing`
