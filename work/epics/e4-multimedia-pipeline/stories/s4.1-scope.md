# S4.1 Scope: Audio Transcription via Whisper

## In Scope

- New edge function `transcribe-audio` (Supabase)
- Fire-and-forget invocation from `evolution-webhook` for audio messages
- Download audio binary via Meta Graph API v21.0 (two-step: media ID -> URL -> binary)
- Transcribe audio via OpenAI Whisper `/v1/audio/transcriptions`
- Update `conversaciones.mensaje` with transcription text
- Invoke `process-samurai-response` with real transcribed text
- Fallback path: update message + generic bot response + activity_logs entry
- Placeholder `[Nota de Voz]` insertion in conversaciones (existing behavior, preserved)

## Out of Scope

- Audio file caching or local storage
- Audio files > 25MB (deferred to S4.3 edge cases)
- Transcription in languages other than Spanish (Whisper auto-detects, no explicit config)
- UI changes to display transcription status
- Video transcription
- Speaker diarization
- Retry logic for Whisper failures (simple fallback only)
- activity_logs observability beyond failure logging (deferred to S4.3)

## Done Criteria

- [ ] Voice note received -> transcribed -> AI responds to actual audio content
- [ ] Fallback works when Whisper API fails (bot asks to repeat, logged in activity_logs)
- [ ] Fallback works when Graph API download fails (same behavior)
- [ ] conversaciones.mensaje updated from placeholder to transcription text
- [ ] Zero regression on text messages (existing flow unchanged)
- [ ] Edge function deploys successfully to Supabase
- [ ] Type-checked (no Pyright/TypeScript errors)

## Dependencies

- OpenAI API key (already available in environment)
- Meta Graph API access via channel api_key (already used by send-message-v3)
- Supabase service role for DB updates (standard for edge functions)

## Risks

| Risk | Mitigation |
|------|------------|
| Whisper latency (5-15s for long audio) | Fire-and-forget pattern; lead sees placeholder immediately |
| Meta token expired | Reuses same api_key renewal as send-message; existing infra handles this |
| Large audio files | Out of scope for S4.1; S4.3 adds size check |
