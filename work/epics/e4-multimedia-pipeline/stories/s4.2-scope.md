# S4.2 Scope: Media Tag Parser

## In Scope

- Regex parser for `<<MEDIA:(https?://[^>]+)>>` pattern in AI response text
- URL extraction from matched media tag
- Clean text generation (AI text with media tag removed)
- Caption mode: send image + caption via send-message-v3 when clean text <= 1024 chars
- Split mode: send image then text separately when clean text > 1024 chars
- Passthrough: no-op when AI response has no media tag

## Out of Scope

- Multiple media tags per response (one tag max for now)
- Media type detection (assume image)
- URL validation or reachability checks (deferred to S4.3)
- Audio/video/document media types
- Media tag generation by AI (prompt engineering)
- activity_logs for media sends (deferred to S4.3)

## Where

- `supabase/functions/process-samurai-response/index.ts` (after line 125, post-AI response)

## Decisions Referenced

- D4: Inline parser in process-samurai-response using regex
- D5: Caption with fallback to separate messages when >1024 chars

## Done Criteria

- [ ] AI response with `<<MEDIA:url>>` results in WhatsApp image delivered to user
- [ ] Clean text sent as caption when <= 1024 chars
- [ ] Clean text sent as separate message when > 1024 chars
- [ ] Messages without media tags work unchanged (zero regression)
- [ ] Tests cover all four scenarios (media+caption, media+split, no-media, edge cases)
