# S4.2 Design: Media Tag Parser in AI Response

## Problem

AI responses containing `<<MEDIA:url>>` tags are sent as literal text to WhatsApp. Customers see raw tags like `<<MEDIA:https://cdn.example.com/poster.jpg>>` instead of actual images. The infrastructure exists (send-message-v3 supports `mediaData`, media_assets table holds URLs) but the AI response flow does not parse or route media tags.

## Value

Bot can send promotional posters and images to customers inline in WhatsApp chat, connecting the existing media infrastructure to the AI response pipeline.

## Approach

**Strategy:** Inline regex parser in `process-samurai-response/index.ts`

After receiving `aiText` from OpenAI (line 125), insert:

1. **Extract** — match `<<MEDIA:(https?://[^>]+)>>` to get the media URL
2. **Clean** — remove the tag from text, trim whitespace
3. **Route** — based on clean text length:
   - **Caption mode** (<=1024 chars): send image with `mediaData` + `message` as caption
   - **Split mode** (>1024 chars): send image alone, then send text as separate message
   - **No tag**: existing flow unchanged

**Components touched:** `process-samurai-response/index.ts` only (modify).

**Decisions from epic:**
- D4: Inline parser using regex (not a separate function)
- D5: Caption with fallback to separate messages when >1024 chars

## Examples

### 1. Caption mode (media + short text)

```
Input:  "Mira este poster <<MEDIA:https://cdn.example.com/poster.jpg>> Te encantara!"
URL:    https://cdn.example.com/poster.jpg
Clean:  "Mira este poster  Te encantara!"
Action: send-message-v3({ mediaData: { url, type: 'image' }, message: cleanText })
```

### 2. Split mode (media + long text)

```
Input:  "<<MEDIA:https://cdn.example.com/menu.jpg>> [1200+ chars of menu description]"
URL:    https://cdn.example.com/menu.jpg
Clean:  "[1200+ chars]"
Action: send-message-v3({ mediaData: { url, type: 'image' }, message: '' })
        send-message-v3({ message: cleanText })
```

### 3. No media tag

```
Input:  "Hola! Como te puedo ayudar?"
URL:    null
Clean:  "Hola! Como te puedo ayudar?"
Action: Existing flow (no change)
```

### 4. Media tag, empty remaining text

```
Input:  "<<MEDIA:https://cdn.example.com/promo.jpg>>"
URL:    https://cdn.example.com/promo.jpg
Clean:  ""
Action: send-message-v3({ mediaData: { url, type: 'image' }, message: '' })
```

## Acceptance Criteria

| ID  | Scenario | Expected |
|-----|----------|----------|
| AC1 | AI response with media tag, clean text <= 1024 chars | Image sent with caption |
| AC2 | AI response with media tag, clean text > 1024 chars | Image without caption + separate text |
| AC3 | AI response without media tag | Current flow unchanged |
| AC4 | Media tag with empty remaining text | Image only, no text message |
| AC5 | Regex extraction | URL correctly parsed from `<<MEDIA:...>>` |

## Out of Scope

- Multiple media tags per response
- Media type detection (assume image)
- URL validation or reachability checks
- activity_logs for media sends (deferred to S4.3)
