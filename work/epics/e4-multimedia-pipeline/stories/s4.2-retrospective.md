# S4.2 Retrospective: Media Tag Parser

## Summary

- **Story:** S4.2 — Media Tag Parser in AI Response
- **Size:** S (estimated) → S (actual)
- **Scope:** Inline regex parser in process-samurai-response + valid_until filter for media_assets

## What Went Well

- Implementación limpia — ~60 líneas de cambio en un solo archivo
- E2E validado en producción: imagen llegó a WhatsApp con caption
- send-message-v3 ya soportaba mediaData — zero cambios necesarios
- El filtro de posters expirados (valid_until) se resolvió en la misma story como bonus

## What To Improve

- El deploy reactivó JWT verification en evolution-webhook, causando 401s de Meta
  - **Aprendizaje:** Siempre deployar webhooks externos con `--no-verify-jwt`
- El primer intento de E2E falló porque el deploy era muy reciente (cold start)
  - **Aprendizaje:** Esperar ~30s después de deploy antes de probar

## Heutagogical Checkpoint

1. **What did I learn?** 
   - Supabase re-deploy puede cambiar la configuración de JWT verification
   - El flag `--no-verify-jwt` es obligatorio para funciones que reciben webhooks externos
   
2. **What would I change?** 
   - Documentar qué funciones necesitan `--no-verify-jwt` en un script de deploy
   
3. **Framework improvements?** 
   - Ninguna — el flujo S4.2 fue directo

4. **What am I more capable of now?** 
   - Supabase Edge Functions deployment con configuración de JWT
   - Media API de WhatsApp (caption mode, split mode)
