# S4.1 Retrospective: Audio Transcription via Whisper

## Summary

- **Story:** S4.1 — Audio Transcription via Whisper
- **Size:** M (estimated) → M (actual)
- **Scope:** New transcribe-audio edge function + evolution-webhook modification

## What Went Well

- Diseño con fire-and-forget probó ser correcto — webhook responde rápido a Meta
- D4 (skip sync AI for audio) fue crucial — sin esto el AI respondería al placeholder
- Whisper transcribió correctamente español desde la primera invocación exitosa
- Fallback logAndFallback() funcionó cuando el token estaba expirado
- Logs en activity_logs dieron visibilidad inmediata del error (401 Meta)

## What To Improve

- El token de desarrollo de Meta expira cada 1-2 horas
  - **Aprendizaje:** Necesitan un System User Token permanente en Meta Business Manager
- El primer intento de E2E usó un token recién expirado
  - **Aprendizaje:** Verificar token antes de probar funcionalidad que depende de Graph API

## Heutagogical Checkpoint

1. **What did I learn?**
   - Meta Graph API media download requiere 2 calls (get URL + download binary)
   - Whisper acepta ogg/opus directo (formato nativo de WhatsApp voice notes)
   - Fire-and-forget con supabase.functions.invoke funciona bien para async

2. **What would I change?**
   - Nada — el flujo async es el correcto para transcripción

3. **Framework improvements?**
   - Ninguna

4. **What am I more capable of now?**
   - OpenAI Whisper API integration
   - Meta Graph API media endpoints
   - Async edge function patterns en Supabase

## Patterns

- PAT: Tokens de desarrollo de Meta expiran en 1-2h. Para producción usar System User Token.
- PAT: Whisper acepta ogg/opus sin conversión. No necesita ffmpeg para WhatsApp voice notes.
