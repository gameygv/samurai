# S4.1 Design: Audio Transcription via Whisper

## Approach

New edge function `transcribe-audio` invoked fire-and-forget from `evolution-webhook` when audio messages arrive. The webhook continues to insert a `[Nota de Voz]` placeholder and respond 200 to Meta immediately. The new function handles audio download, transcription, DB update, and AI invocation asynchronously.

## Components

### NEW: supabase/functions/transcribe-audio/index.ts

**Input contract:**
```typescript
{
  media_id: string;      // msg.audio.id from Meta webhook
  lead_id: string;       // UUID of the lead
  message_id: string;    // wamid (msg.id) stored in conversaciones
  channel_id: string;    // UUID of the channel (for api_key lookup)
}
```

**Internal flow:**

1. **Get channel api_key** â€” query `whatsapp_channels` by `channel_id` to get `api_key`
2. **Get OpenAI api_key** â€” query `app_config` where `key = 'openai_api_key'`
3. **Get audio URL** â€” `GET https://graph.facebook.com/v21.0/{media_id}` with `Bearer {channel.api_key}` returns `{ url }`
4. **Download audio binary** â€” `GET {url}` with `Bearer {channel.api_key}` returns ogg/opus binary
5. **Transcribe** â€” `POST https://api.openai.com/v1/audio/transcriptions` with FormData: `file` (audio.ogg), `model` (whisper-1), `language` (es)
6. **Update conversaciones** â€” `UPDATE conversaciones SET mensaje = '[TRANSCRIPCION DE NOTA DE VOZ]: "{text}"' WHERE message_id = {message_id}`
7. **Invoke AI** â€” call `process-samurai-response` with `{ lead_id, client_message: text }`

**Fallback (any failure at steps 3-5):**

1. `UPDATE conversaciones SET mensaje = '[Nota de Voz â€” no se pudo transcribir]' WHERE message_id = {message_id}`
2. Invoke `process-samurai-response` with `{ lead_id, client_message: '[Nota de Voz â€” no se pudo transcribir]' }`
3. Log error to `activity_logs`

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Where to transcribe | Separate edge function `transcribe-audio` | Decouples latency from webhook; fire-and-forget keeps webhook fast |
| D2 | How to download audio | Two Graph API calls (get URL + download binary) | Meta requires media ID -> URL -> binary flow |
| D3 | Fallback strategy | Generic bot response + activity_log | Simple, observable, no silent failures |
| D4 | Skip sync AI for audio | Webhook skips process-samurai-response when audio | Avoids AI responding to placeholder; transcribe-audio calls it with real text |

## External Integrations

### Meta Graph API v21.0
- **Auth:** `Bearer {channel.api_key}` (same token used by send-message-v3)
- **Call 1:** `GET https://graph.facebook.com/v21.0/{media_id}` â€” returns `{ url }` (temporary download URL)
- **Call 2:** `GET {url}` â€” returns audio binary (ogg/opus format)
- **Failure modes:** 401 (expired token), 404 (media expired after 30 days), network timeout

### OpenAI Whisper
- **Auth:** `Bearer {openai_api_key}` from `app_config` table
- **Call:** `POST https://api.openai.com/v1/audio/transcriptions`
- **Body:** FormData with `file` (blob, named audio.ogg), `model` (whisper-1), `language` (es)
- **Response:** `{ text: "transcribed content" }`
- **Failure modes:** 429 (rate limit), 413 (file too large > 25MB), network timeout

## Example Flows

### 1. Success
```
Lead sends voice note: "Hola quiero info de talleres"
  -> evolution-webhook: inserts "[Nota de Voz]" in conversaciones
  -> fires transcribe-audio (does NOT call process-samurai-response)
  -> transcribe-audio: downloads audio, transcribes
  -> UPDATE conversaciones: '[TRANSCRIPCION DE NOTA DE VOZ]: "Hola quiero info de talleres"'
  -> Calls process-samurai-response -> AI responds to actual content
```

### 2. Whisper Failure
```
Lead sends voice note
  -> evolution-webhook: inserts "[Nota de Voz]", fires transcribe-audio
  -> transcribe-audio: downloads audio OK, Whisper returns error
  -> UPDATE conversaciones: '[Nota de Voz â€” no se pudo transcribir]'
  -> Calls process-samurai-response -> AI responds "No pude escuchar tu audio"
  -> Logs error in activity_logs
```

### 3. Meta Download Failure
```
Lead sends voice note
  -> evolution-webhook: inserts "[Nota de Voz]", fires transcribe-audio
  -> transcribe-audio: Graph API returns 401 (token issue)
  -> Same fallback as Whisper failure
```

## Acceptance Criteria

3 scenarios from s4.1-story.md:
1. **Successful voice note transcription** â€” full flow from audio to AI response on real content
2. **Whisper transcription fails** â€” fallback message, bot asks to repeat, logged
3. **Text messages unaffected** â€” no call to transcribe-audio for non-audio messages
