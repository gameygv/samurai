## T1: Create transcribe-audio edge function

**File:** `supabase/functions/transcribe-audio/index.ts` (NEW)

**What:**
- Create new Supabase edge function accepting `{ media_id, lead_id, message_id, channel_id }`
- Query `whatsapp_channels` by `channel_id` to get `api_key`
- Query `app_config` where `key = 'openai_api_key'` to get OpenAI key
- Call Meta Graph API: `GET https://graph.facebook.com/v21.0/{media_id}` with channel Bearer token to get temporary URL
- Download audio binary: `GET {url}` with channel Bearer token
- Transcribe: `POST https://api.openai.com/v1/audio/transcriptions` with FormData (`file`: audio.ogg, `model`: whisper-1, `language`: es)
- Update conversaciones: `SET mensaje = '[TRANSCRIPCION DE NOTA DE VOZ]: "{text}"' WHERE message_id = {message_id}`
- Invoke `process-samurai-response` with `{ lead_id, client_message: text }`

**Fallback (any error at Graph API or Whisper steps):**
- Update conversaciones: `SET mensaje = '[Nota de Voz — no se pudo transcribir]' WHERE message_id = {message_id}`
- Invoke `process-samurai-response` with `{ lead_id, client_message: '[Nota de Voz — no se pudo transcribir]' }`
- Insert error entry in `activity_logs`

**Deploy:** `supabase functions deploy transcribe-audio --no-verify-jwt`

**Verify:** Function deploys without error.

## T2: Modify evolution-webhook to fire transcribe-audio for audio

**File:** `supabase/functions/evolution-webhook/index.ts` (MODIFY)

**Changes:**

1. **Extract audioMediaId** (around line 76-77): After `text = '[Nota de Voz]'`, add `audioMediaId = msg.audio?.id || null;`. Declare `let audioMediaId: string | null = null;` at top of message parsing block.

2. **Fire-and-forget transcribe-audio** (after conversaciones insert, around line 166): When `audioMediaId` is set, invoke `transcribe-audio` via `supabase.functions.invoke()` with `{ media_id: audioMediaId, lead_id: lead.id, message_id: messageId, channel_id: actualChannelId }`. Use `.catch()` to swallow errors (fire-and-forget pattern, mirrors existing `analyze-leads` call at line 189).

3. **Skip synchronous process-samurai-response for audio** (D4): Wrap the existing `process-samurai-response` call (lines 172-183) in `if (!audioMediaId)` guard. The transcribe-audio function handles AI invocation after transcription completes, preventing the bot from responding to the `[Nota de Voz]` placeholder.

**Deploy:** `supabase functions deploy evolution-webhook --no-verify-jwt`

**Verify:** Function deploys without error.

## Risk Mitigations

| Risk | Mitigation |
|------|------------|
| Whisper latency (5-15s) | Fire-and-forget pattern; lead sees placeholder immediately |
| Meta token expired | Reuses same api_key used by send-message-v3 |
| Large audio (>25MB) | Out of scope for S4.1; deferred to S4.3 |
