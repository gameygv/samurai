# S4.1: Audio Transcription via Whisper

## User Story (Connextra)

**As a** lead sending voice notes via WhatsApp,
**I want** my audio messages to be transcribed and understood by the bot,
**So that** I get relevant AI responses to what I actually said, not a generic placeholder.

## Where

- New edge function: `transcribe-audio`
- Modified edge function: `evolution-webhook`

## What

1. `evolution-webhook` detects `msg.type === 'audio'`, extracts `msg.audio.id`
2. Inserts placeholder `[Nota de Voz]` in conversaciones (as today)
3. Fire-and-forget call to `transcribe-audio` with `{ media_id, lead_id, message_id, channel_id }`
4. `transcribe-audio`:
   - Gets `api_key` from channel via `channel_id`
   - GET Graph API `v21.0/{media_id}` for temporary download URL
   - GET temporary URL for binary audio data
   - POST binary to OpenAI Whisper `/v1/audio/transcriptions`
5. UPDATE conversaciones SET mensaje = `[TRANSCRIPCION DE NOTA DE VOZ]: "texto"`
6. Invokes `process-samurai-response` with transcribed text

## Fallback

If Whisper fails at any step:
1. UPDATE conversaciones SET mensaje = `[Nota de Voz — no se pudo transcribir]`
2. Invoke `process-samurai-response` so bot asks the lead to repeat their message
3. Log failure details in `activity_logs`

## Acceptance Criteria (Gherkin)

### Scenario: Successful voice note transcription
```gherkin
Given a lead sends a voice note via WhatsApp
When evolution-webhook receives the audio message
Then it inserts "[Nota de Voz]" as placeholder in conversaciones
And it fires-and-forgets to transcribe-audio with media_id, lead_id, message_id, channel_id
When transcribe-audio receives the request
Then it retrieves the api_key from the channel
And it calls Graph API v21.0/{media_id} to get the temporary URL
And it downloads the audio binary from the temporary URL
And it sends the binary to Whisper /v1/audio/transcriptions
And it updates conversaciones.mensaje to '[TRANSCRIPCION DE NOTA DE VOZ]: "{text}"'
And it invokes process-samurai-response with the transcribed text
```

### Scenario: Whisper transcription fails
```gherkin
Given a lead sends a voice note via WhatsApp
And the voice note has been forwarded to transcribe-audio
When Whisper API returns an error or times out
Then transcribe-audio updates conversaciones.mensaje to '[Nota de Voz — no se pudo transcribir]'
And it invokes process-samurai-response so the bot asks the lead to repeat
And it logs the failure in activity_logs
```

### Scenario: Text messages are unaffected
```gherkin
Given a lead sends a regular text message via WhatsApp
When evolution-webhook processes the message
Then the message flows through the existing pipeline unchanged
And no call to transcribe-audio is made
```

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Where to transcribe | Separate edge function `transcribe-audio` | Decouples latency from webhook response; fire-and-forget keeps webhook fast |
| D2 | How to download audio | Two Graph API calls (get URL + download binary) | Meta requires media ID -> URL -> binary flow |
| D3 | Fallback strategy | Generic response — bot asks to repeat + log | Simple, observable, no silent failures |
