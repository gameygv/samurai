# Branch Protection Guide — `main`

Branch protection cannot be configured from code. Follow these manual steps in GitHub.

## Steps

1. Go to **Settings > Branches** in the repository
2. Click **Add branch protection rule**
3. Set **Branch name pattern** to `main`
4. Enable **Require a pull request before merging**
5. Enable **Require status checks to pass before merging**
6. Search and select the **ci** status check
7. Enable **Require branches to be up to date before merging**
8. Click **Create** (or **Save changes**)

## Result

After configuration, PRs targeting `main` cannot be merged unless the CI workflow passes. Direct pushes to `main` are blocked for non-admins.
