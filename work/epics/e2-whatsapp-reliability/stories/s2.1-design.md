# S2.1: Design — Consolidar inserción de mensajes IA

## Changes

### 1. process-samurai-response/index.ts

**Replace** PostgREST direct insert (lines 129-155) with supabase client:
```ts
const { error: convError } = await supabase.from('conversaciones').insert({
    lead_id: lead.id, emisor: 'IA', mensaje: aiText, platform: 'WHATSAPP'
}).select();
```
Keep error logging to activity_logs on failure. Continue sending to WhatsApp regardless (D1: option A).

**Keep** `{ aiText }` in response body (D2: option A).

### 2. evolution-webhook/index.ts

**Remove** lines 151-155 (duplicate IA insert):
```ts
// DELETE THIS:
if (fnData.aiText) {
    await supabase.from('conversaciones').insert({
        lead_id: lead.id, emisor: 'IA', mensaje: fnData.aiText, platform: 'WHATSAPP'
    });
}
```

## Design Decisions

| ID | Decision | Choice | Rationale |
|----|----------|--------|-----------|
| S2.1-D1 | Behavior on insert failure | Log + continue sending | Insert failure near impossible with SERVICE_ROLE_KEY; don't sacrifice UX |
| S2.1-D2 | Response body format | Keep { aiText } | Backwards compatible, useful for debugging, zero cost |

## Not Changed

- send-message-v3 — no changes
- Frontend — no changes
- Database schema — no migration
