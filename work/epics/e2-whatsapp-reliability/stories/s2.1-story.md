# S2.1: Consolidar inserción de mensajes IA

## User Story

Como operador del CRM, quiero que los mensajes generados por la IA se guarden
exactamente una vez en la tabla conversaciones, para que el historial de chat
sea confiable y sin duplicados.

## Context

Actualmente hay dos puntos de insert para mensajes IA:
1. `process-samurai-response/index.ts:129-140` — PostgREST directo (hack)
2. `evolution-webhook/index.ts:151-155` — supabase client después de recibir response

Esto causa duplicados o pérdidas dependiendo de timing y errores.

## Acceptance Criteria

- [ ] Solo `process-samurai-response` inserta mensajes IA en conversaciones
- [ ] Usa cliente supabase estándar (.insert().select()), no PostgREST directo
- [ ] El insert redundante en evolution-webhook está eliminado
- [ ] El response de process-samurai-response sigue retornando { aiText } para compatibilidad
- [ ] Tests verifican el flujo
