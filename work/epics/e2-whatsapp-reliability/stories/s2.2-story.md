# S2.2: Deduplicación por wamid en conversaciones

## User Story

Como operador del CRM, quiero que cada mensaje de WhatsApp tenga un ID único (wamid)
almacenado, para que si Meta reenvía un webhook el mensaje no se duplique en el historial.

## Context

Meta asigna un `wamid` a cada mensaje (entrante y saliente). Hoy no se almacena.
Si Meta reenvía un webhook (por timeout), el mismo mensaje se procesa dos veces.
Además, send-message-v3 no retorna el wamid del mensaje enviado.

Design decisions from epic:
- D3: Columna message_id con UNIQUE parcial (WHERE NOT NULL)
- D4: Columna delivery_status para status webhooks (preparación para S2.3)
- D5: Actualizar Graph API a v21.0 (bundled con esta story)

## Acceptance Criteria

- [ ] Columna message_id (TEXT, nullable) en conversaciones
- [ ] Unique partial index: solo WHERE message_id IS NOT NULL
- [ ] Columna delivery_status (TEXT, nullable) en conversaciones
- [ ] evolution-webhook guarda wamid en mensajes entrantes (CLIENTE)
- [ ] send-message-v3 parsea y retorna wamid del response de Meta
- [ ] process-samurai-response guarda wamid en mensajes salientes (IA)
- [ ] Graph API actualizada a v21.0 (S2.4 bundled)
- [ ] Inserts existentes sin wamid (Panel, campañas) siguen funcionando
