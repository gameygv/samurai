# S2.3: Procesar status webhooks de Meta

## User Story

Como operador del CRM, quiero ver el estado de entrega de los mensajes
enviados por el bot (sent/delivered/read), para saber si el cliente
realmente recibió la respuesta.

## Context

Meta envía webhooks de status cuando un mensaje cambia de estado:
sent → delivered → read. Hoy el webhook los descarta en línea 34:
`if (!change || change.statuses) return new Response('ok', { status: 200 });`

S2.2 ya agregó:
- Columna `message_id` (wamid) en conversaciones
- Columna `delivery_status` en conversaciones

Solo falta procesar los status webhooks y hacer UPDATE.

## Acceptance Criteria

- [ ] Status webhooks de Meta actualizan delivery_status en conversaciones
- [ ] Solo actualiza si el wamid existe en conversaciones
- [ ] Statuses soportados: sent, delivered, read, failed
- [ ] No rompe el flujo de mensajes entrantes
- [ ] Webhook retorna 200 rápido (no bloquea)
