# S8.4: Playwright e2e — Design

## Gemba (current state)

- Zero e2e tests exist
- App runs on `localhost:8080` via Vite dev server
- Auth via Supabase Auth (email/password), session stored in localStorage
- Routes: `/login` (public), `/` (dashboard), `/contacts`, `/leads`, `/settings` (private, admin-only for Settings)
- Login page: email + password form, submits via `supabase.auth.signInWithPassword()`
- Invalid login shows toast error: "Email o contrasena incorrectos."
- Successful login navigates to `/`

## Playwright Configuration

### `playwright.config.ts` (project root)

- **baseURL**: `http://localhost:8080`
- **Browser**: Chromium only (fastest for CI; add Firefox/WebKit later)
- **Test directory**: `e2e/`
- **Web server**: Auto-start Vite dev server before tests
  ```
  webServer: {
    command: 'npm run dev',
    url: 'http://localhost:8080',
    reuseExistingServer: !process.env.CI
  }
  ```
- **Projects**:
  1. `setup` — runs `auth.setup.ts`, no dependencies
  2. `chromium` — runs all `*.spec.ts`, depends on `setup`

### Auth state directory

- `e2e/.auth/` — gitignored, contains saved auth state (`user.json`)

## Auth Strategy

### Problem
Login per test is slow and flaky. Supabase auth returns a session token.

### Solution
- `e2e/auth.setup.ts` logs in once with `E2E_USER_EMAIL` / `E2E_USER_PASSWORD` env vars
- Saves browser storage state to `e2e/.auth/user.json`
- All spec files use `storageState: 'e2e/.auth/user.json'` via project config
- Login spec does NOT use saved state (tests the login page itself)

### Credentials
- Use env vars: `E2E_USER_EMAIL`, `E2E_USER_PASSWORD`
- Local dev: `.env.test` file (gitignored) with credentials
- CI: set as secrets (Epic 10)

## Test Files

### `e2e/auth.setup.ts`
- Navigate to `/login`
- Fill email and password from env vars
- Click submit button
- Wait for navigation to `/` (dashboard)
- Save storage state to `e2e/.auth/user.json`

### `e2e/login.spec.ts` (NO saved auth state)
- **Test: valid login** — fill creds, submit, verify redirect to dashboard
- **Test: invalid login** — fill wrong password, submit, verify error toast appears
- **Test: redirect if authenticated** — with saved state, navigate to `/login`, verify redirect to `/`

### `e2e/dashboard.spec.ts` (uses saved auth state)
- **Test: page loads** — navigate to `/`, verify page title or heading
- **Test: stat cards visible** — verify stat card elements are rendered (by role or test-id)
- **Test: layout renders** — sidebar/nav visible

### `e2e/navigation.spec.ts` (uses saved auth state)
- **Test: contacts page** — click Contacts nav link, verify URL is `/contacts`, page renders
- **Test: leads page** — click Leads nav link, verify URL is `/leads`, page renders
- **Test: settings page** — click Settings nav link, verify URL is `/settings`, page renders

### `e2e/contacts.spec.ts` (uses saved auth state)
- **Test: contacts list renders** — navigate to `/contacts`, verify list/table is visible
- **Test: open contact chat** — click first contact row, verify chat panel opens

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Tests run against localhost:8080 | Real Supabase backend, real data. Avoids mock complexity. |
| Playwright auto-starts Vite via `webServer` | No manual server startup needed. |
| Auth state persisted to file | Login once, reuse everywhere. Fast. |
| Smoke tests only | Verify pages load and key elements exist, not full business logic. |
| No fixtures/seed data | Tests check UI rendering, not specific data content. |
| Chromium only | Fastest. Multi-browser adds CI time with little value for smoke tests. |
| Env vars for credentials | No hardcoded secrets. `.env.test` gitignored. |


## Risks

| Risk | Mitigation |
|------|------------|
| Tests flaky due to real Supabase data | Smoke tests only check element presence, not data values |
| Auth token expires during test run | Playwright session is short-lived; token valid for hours |
| Vite dev server slow to start | `reuseExistingServer` for local dev; CI starts fresh |
