# S8.2: Tipos Supabase + eliminar @ts-nocheck — Scope

## Objective

Generate TypeScript types from Supabase schema, create shared types for Edge Functions, and remove `@ts-nocheck` from 5 critical functions to establish type safety in the backend.

## In Scope

- Run `supabase gen types typescript --linked` to generate `src/integrations/supabase/types.ts`
- Update `src/integrations/supabase/client.ts` to export a typed Supabase client (`createClient<Database>`)
- Create `supabase/functions/_shared/types.ts` with Database type for Edge Functions (Deno-compatible)
- Remove `@ts-nocheck` from 5 Edge Functions and fix resulting type errors:
  1. `send-message-v3`
  2. `analyze-leads`
  3. `get-samurai-context`
  4. `process-samurai-response`
  5. `evolution-webhook`

## Out of Scope

- Removing `@ts-nocheck` from all 30 Edge Functions (only 5 targeted)
- Fixing all `any` types in frontend code
- Enabling `strict: true` in tsconfig
- Refactoring business logic in Edge Functions
- CI/CD pipeline changes

## Acceptance Criteria

1. `src/integrations/supabase/types.ts` exists (generated from remote schema)
2. `client.ts` exports a typed Supabase client using `Database` generic
3. `supabase/functions/_shared/types.ts` exists with Database type for Deno imports
4. 5 Edge Functions compile without `@ts-nocheck` (`deno check` passes)
5. `npm test` passes (56 tests)
6. `npm run build` passes
