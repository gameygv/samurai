# S8.1: Vitest Setup + Tests Frontend — Scope

## Objective

Configure Vitest with React Testing Library as the frontend test runner and write initial tests covering utils, lib helpers, and custom hooks to establish the `npm test` walking skeleton.

## In Scope

- Install and configure Vitest + @testing-library/react + jsdom
- Add `test` script to package.json
- Write tests for utility functions in `src/utils/` or `src/lib/`
- Write tests for custom hooks
- Minimum 5 passing tests
- Vitest config file (vitest.config.ts)

## Out of Scope

- Backend/Edge Function tests (S8.3)
- E2E/Playwright tests (S8.4)
- Supabase type generation (S8.2)
- CI/CD integration (E10)
- Component rendering tests beyond hooks

## Acceptance Criteria

1. `npm test` runs and exits with code 0
2. At least 5 tests pass covering utils, lib, or hooks
3. Vitest config is committed and functional
4. No changes to production source code
5. `test` script exists in package.json
