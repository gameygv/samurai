# S8.4: Playwright e2e — Scope

## Objective

Configure Playwright and write e2e smoke tests for critical business flows, validating that login, dashboard, navigation, and contacts work end-to-end in SAMURAI CRM.

## In Scope

- Playwright installation and configuration (`playwright.config.ts`)
- Auth state persistence (login once, reuse across tests)
- Test files:
  1. `e2e/auth.setup.ts` — login and save auth state
  2. `e2e/login.spec.ts` — valid credentials -> dashboard, invalid -> error
  3. `e2e/dashboard.spec.ts` — dashboard loads with stat cards
  4. `e2e/navigation.spec.ts` — navigate to Contacts, Leads, Settings
  5. `e2e/contacts.spec.ts` — contacts list renders, click to open chat
- Script `test:e2e` in package.json
- Environment variable based credentials (`.env.test`, gitignored)

## Out of Scope

- Testing WhatsApp message sending (requires external API)
- Testing cron functions
- Visual regression testing
- Multi-browser testing (chromium only for now)
- CI/CD pipeline integration (Epic 10)
- Seed data / fixtures (smoke tests only — verify UI renders, not specific data)

## Acceptance Criteria

1. Playwright installed and configured
2. `npx playwright test` runs and passes
3. Login flow tested (valid credentials -> dashboard, invalid -> error)
4. Dashboard loads and shows stat cards
5. Navigation to key pages works (Contacts, Leads, Settings)
6. `npm test` (vitest) still passes
