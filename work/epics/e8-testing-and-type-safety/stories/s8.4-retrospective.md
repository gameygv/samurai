# S8.4 Retrospective — Playwright e2e Tests

## What Went Well

- 12 e2e tests passing across 4 spec files + 1 auth setup
- Auth state persistence pattern: login once in setup, reuse saved state across all test projects — fast and reliable
- Auto webServer config: Playwright starts `npm run dev` automatically, no manual setup needed
- Resilient selectors throughout: `getByRole`, `getByLabel`, `getByText`, `getByPlaceholder` — no fragile CSS selectors
- Credentials externalized to `.env.test` (gitignored), zero secrets in tracked files
- Existing vitest suite (56 tests) and build remain green — no regressions

## What Could Improve

- Tests are smoke-level only — they verify pages render and navigate, but do not assert specific data values
- No data fixtures or seed data — tests depend on whatever data exists in the environment
- Navigation spec tests Leads and Pipeline but not Settings (scope said Settings, implementation used Pipeline)

## Patterns Identified

- **Auth state persistence**: Save browser storage state after login setup, inject into test projects via `storageState`. Eliminates repeated login overhead and keeps tests fast.
- **WebServer auto-start**: Playwright's `webServer` config starts the dev server automatically with health check. Reduces manual setup and makes CI integration straightforward.
- **Login spec isolation**: Login tests explicitly clear storage state (`storageState: { cookies: [], origins: [] }`) to test the login page without auth leakage from setup.
