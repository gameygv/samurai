# S8.3 Retrospective — Deno Tests for Backend Edge Functions

## What Went Well

- 82 Deno tests across 7 test files covering all 5 critical Edge Functions
- Zero production files modified — tests are purely additive
- Tests validate real business logic patterns: COMPRADO blindaje, PERDIDO rescue, CAPI trigger conditions, intent validation, webhook parsing (Meta + Gowa), auth middleware, CORS, context assembly
- All tests pass in under 1 second (659ms total)
- Existing vitest suite (56 tests) and build remain green — no regressions

## What Could Improve

- Tests replicate logic patterns rather than importing actual function code directly
- This is a known limitation: Edge Functions use `Deno.serve()` as a top-level side effect, making direct imports execute the server
- Pattern-based testing validates the logic but not the exact code path in production

## Patterns Identified

- **Pattern-based testing**: When modules have side effects at import time (e.g., `Deno.serve()`), replicate the core logic in test files and validate patterns. This is a pragmatic alternative to import-based testing that still catches logic bugs.
- **Business rule coverage**: Prioritize tests for business-critical rules (COMPRADO skip, PERDIDO rescue, CAPI firing conditions) over infrastructure plumbing.
