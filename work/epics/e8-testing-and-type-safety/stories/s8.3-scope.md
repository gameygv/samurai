# S8.3: Deno test + tests backend — Scope

## Objective

Write Deno tests for 5 critical Edge Functions with mocked Supabase client and fetch, establishing backend test coverage for SAMURAI's core business logic.

## In Scope

- Test files for 5 Edge Functions:
  1. `send-message-v3` (135 LOC)
  2. `analyze-leads` (224 LOC)
  3. `get-samurai-context` (175 LOC)
  4. `process-samurai-response` (249 LOC)
  5. `evolution-webhook` (230 LOC)
- Shared test helpers for mocking:
  - Mock Supabase client factory (chainable `.from().select().eq()` etc.)
  - Mock fetch for OpenAI API calls
  - Mock `serve()` to capture handler without starting HTTP server
- Tests cover happy path + error handling per function

## Out of Scope

- Testing all 30 Edge Functions (only 5 targeted)
- Integration tests against real Supabase DB
- Integration tests against real OpenAI API
- Refactoring production code beyond minimal extraction
- CI/CD pipeline integration (Epic 10)
- Performance/load testing

## Acceptance Criteria

1. Each of the 5 functions has a test file (`<name>.test.ts`)
2. Tests cover happy path + error handling
3. Tests run with `deno test` command and pass
4. Supabase client is mocked (no real DB calls)
5. OpenAI fetch is mocked (no real API calls)
6. `npm test` (frontend) still passes
