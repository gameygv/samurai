# S9.2 Plan: RLS Policies Reales por Rol

## Tasks

### T1: Create helper function `auth.get_user_role()`

- **ID**: T1
- **Description**: Create a Supabase SQL migration that defines `auth.get_user_role()` as a `SECURITY DEFINER`, `STABLE` function returning `TEXT`. The function reads `public.profiles.role` for `auth.uid()`, bypassing RLS to avoid circular dependency. File: `supabase/migrations/20260407120000_create_get_user_role.sql`.
- **Verification**: Function exists in `auth` schema; `SELECT auth.get_user_role()` returns a role string when called with a valid JWT.
- **Dependencies**: None.

### T2: Create forward RLS migration

- **ID**: T2
- **Description**: Create `supabase/migrations/20260407120001_rls_real_policies.sql` that:
  1. Drops all existing `USING(true)` policies on: leads, conversaciones, contacts, profiles, media_assets, whatsapp_channels, app_config, credit_sales, credit_installments, activity_logs, errores_ia, frases_geoffrey, prompt_versions, versiones_prompts_aprendidas, historial_corregiria, followup_config, followup_history, knowledge_documents, main_website_content, meta_capi_events, agent_evaluations.
  2. Creates new role-based policies per the design's access model (admin/dev full, gerente full read, agent scoped to `assigned_to = auth.uid()` chain).
  3. `REVOKE ALL ON ALL TABLES IN SCHEMA public FROM anon`.
  4. `GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO authenticated`.
  5. `CREATE INDEX IF NOT EXISTS idx_leads_assigned_to ON public.leads(assigned_to)`.
  6. `NOTIFY pgrst, 'reload config'` and `NOTIFY pgrst, 'reload schema'`.
  7. Entire migration wrapped in `BEGIN/COMMIT`.
- **Verification**: Migration file is syntactically valid SQL; all policy names match design document.
- **Dependencies**: T1 (helper function must exist before policies reference it).

### T3: Create rollback migration

- **ID**: T3
- **Description**: Create `supabase/migrations/20260407120002_rls_rollback.sql` that:
  1. Drops `auth.get_user_role()` function.
  2. Drops all new role-based policies created in T2.
  3. Recreates original `USING(true)` policies with their original names.
  4. Re-grants anon access: `GRANT ALL ON ALL TABLES IN SCHEMA public TO anon`.
  5. Notifies PostgREST to reload.
- **Verification**: Rollback file exists; can be applied to restore previous permissive state.
- **Dependencies**: T2 (must know exact policy names to drop).

### T4: Verify frontend build still passes

- **ID**: T4
- **Description**: Run `npm test` (expect 56 passing tests) and `npm run build` to confirm no frontend breakage. These are SQL-only migrations with zero frontend changes, so all tests should remain green.
- **Verification**: `npm test` passes all 56 tests; `npm run build` exits 0.
- **Dependencies**: T2, T3 (migrations must be written before final verification).

### T5: Document verification queries
