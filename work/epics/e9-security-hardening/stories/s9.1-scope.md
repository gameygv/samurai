# S9.1 Scope — Auth middleware + proteger funciones admin

## Objective

Create `_shared/auth.ts` with `requireAuth()` and `requireRole()` helpers. Integrate in 6 admin Edge Functions to enforce JWT authentication and admin role verification.

## In Scope

- `_shared/auth.ts` helper module with `requireAuth()` and `requireRole()`
- Protect 6 admin functions:
  - system-wipe
  - admin-create-user
  - admin-reset-password
  - manage-auth-users
  - update-user-email
  - manage-prompt-versions

## Out of Scope

- Webhook/cron functions (evolution-webhook, process-followups, etc.)
- RLS policies (S9.2)
- Secrets migration (S9.3)

## Acceptance Criteria

1. `_shared/auth.ts` exports `requireAuth()` and `requireRole()`
2. `system-wipe` returns 401 without JWT
3. `system-wipe` returns 403 with non-admin JWT
4. `system-wipe` works normally with admin JWT
5. All 6 admin functions protected with JWT + role verification
6. `evolution-webhook` still works without JWT (not affected)
