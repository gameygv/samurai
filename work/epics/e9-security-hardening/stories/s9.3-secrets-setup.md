# Secrets Setup for SAMURAI

Run these commands to configure Edge Function secrets:

```bash
# OpenAI
supabase secrets set OPENAI_API_KEY="<your-openai-api-key>"

# Meta CAPI
supabase secrets set META_ACCESS_TOKEN="<your-meta-access-token>"
supabase secrets set META_PIXEL_ID="<your-meta-pixel-id>"
```

## Deploy Order

1. Set secrets first: `supabase secrets set ...`
2. Deploy functions: `supabase functions deploy`
3. Verify in production

## Verification

After setting secrets and deploying functions:
1. Send a test message to trigger process-samurai-response -> AI should respond
2. Check analyze-leads logs -> should use env var
3. Test meta-capi-sender -> should work without frontend sending token

## Fallback Behavior

All functions use the pattern:
```ts
const apiKey = Deno.env.get('OPENAI_API_KEY') || configMap['openai_api_key'];
```

If the env var is not set, the function falls back to the app_config value.
The app_config rows are NOT deleted — they serve as a safety net.
