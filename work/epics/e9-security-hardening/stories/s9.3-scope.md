# S9.3: Secrets to Env Vars — Scope

## Objective

Move API secrets from `app_config` DB table to Supabase Edge Function environment variables (`Deno.env.get`). Eliminate secret transmission through the frontend.

## In Scope

- `openai_api_key` -> `OPENAI_API_KEY` env var in all 10 functions that read it:
  process-samurai-response, analyze-leads, get-ai-suggestions, simulate-samurai,
  evaluate-agent, audit-chat-performance, tune-samurai-prompts, scrape-website,
  transcribe-audio, analyze-receipt
- `meta_access_token` -> `META_ACCESS_TOKEN` env var in: analyze-leads, meta-capi-sender
- `meta_pixel_id` -> `META_PIXEL_ID` env var in: analyze-leads, meta-capi-sender
- Frontend cleanup: Payments.tsx stops sending `meta_access_token` in request body
- Frontend cleanup: MetaVerify.tsx stops sending `accessToken` in request body
- Documentation of `supabase secrets set` commands

## Out of Scope

- Supabase Vault
- Secret rotation
- CI/CD integration
- evolution_api_key, wc_consumer_key, wc_consumer_secret (future story)
- Removing secret rows from app_config (deferred until env vars verified in production)

## Acceptance Criteria

1. All Edge Functions read OpenAI key from `Deno.env.get('OPENAI_API_KEY')` with fallback to app_config
2. Meta access token read from `Deno.env.get('META_ACCESS_TOKEN')` with fallback
3. Meta pixel ID read from `Deno.env.get('META_PIXEL_ID')` with fallback
4. Frontend does NOT read or transmit `meta_access_token`
5. `npm test` passes, `npm run build` passes
6. A command list for `supabase secrets set` is documented
