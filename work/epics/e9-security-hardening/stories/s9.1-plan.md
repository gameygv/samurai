# S9.1 Plan — Auth middleware + proteger funciones admin

## Overview

8 tasks: create shared auth helper, integrate in 6 admin functions, verify build.

## Tasks

### T1: Create `_shared/auth.ts` with requireAuth() and requireRole()

- **ID**: T1
- **Description**: Create `supabase/functions/_shared/auth.ts` implementing two exported functions:
  - `requireAuth(req)`: extract Bearer token from Authorization header, validate JWT with anon-key client via `auth.getUser(token)`, lookup `profiles.role` with service_role client. Return `{ data: { user, profile }, error: null }` on success or `{ data: null, error: Response }` (401) on failure.
  - `requireRole(req, allowedRoles)`: call requireAuth, then check `profile.role` is in `allowedRoles` array. Return 403 Response if role not allowed.
- **Dependencies**: none
- **Verification**: file exists at `supabase/functions/_shared/auth.ts`, exports `requireAuth` and `requireRole`, TypeScript compiles without errors.
