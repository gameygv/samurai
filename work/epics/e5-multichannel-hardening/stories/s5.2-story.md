# S5.2: Auto-Routing por Ciudad

**As** a manager
**I want** leads to be automatically assigned to the agent responsible for their city
**So that** I don't have to manually assign every new lead

## Context

- analyze-leads already extracts ciudad from conversation
- profiles.territories stores city arrays per agent
- app_config channel_routing_mode = 'auto' enables this feature
- D3: Routing triggers when analyze-leads detects city
- D4: Exact case-insensitive match + AI fallback for fuzzy matching

## AC

Given routing_mode = 'auto' and lead mentions "Monterrey"
When analyze-leads extracts ciudad = "Monterrey"
And agent Carlos has territories = ['Monterrey', 'Saltillo']
Then lead.assigned_to = Carlos.id
And activity_log records the assignment

Given routing_mode = 'auto' and lead mentions "Mazatlán"
When no agent has exact match for "Mazatlán"
Then AI determines closest agent territory
And lead.assigned_to = closest_agent.id

Given routing_mode = 'channel' (not auto)
When analyze-leads extracts ciudad
Then no auto-routing happens (existing behavior)

Given lead already has assigned_to
When analyze-leads detects city
Then do NOT reassign (respect manual assignment)
