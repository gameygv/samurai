# S11.2 Design: Expediente Maestro — 3 Tabs

## Component Strategy

**Refactor** `src/components/contacts/EditContactDialog.tsx` — not a new component. The existing dialog already has 3 tabs and the dark theme. We enhance it with the new profile fields and attendance toggle.

## Current State Analysis

The existing `EditContactDialog.tsx` already has:
- 3 tabs: "perfil" (Datos Base), "academia" (Ficha Curricular), "notas" (Notas del Perfil)
- Dialog title: "Expediente Maestro del Cliente" with subtitle
- Dark theme: bg-[#0f0f11], indigo/amber accents
- Academic record JSONB array with add/remove
- Internal notes JSONB array with add/remove
- "Extraer del Chat" button for AI analysis
- Catalog fetch from app_config (courses, locations, teachers)

**What's missing:**
1. Profile fields: dieta, alimentacion, alergias, motivo_curso (not in formData, not loaded, not saved)
2. Attendance toggle per academic record entry
3. Nivel field per academic record entry

## Tab Structure

### Tab 1 — Datos Base (value="perfil")
No structural changes needed. Already well-organized with:
- Section "DATOS PERSONALES": nombre, apellido (2-col), telefono, email (2-col)
- Section "UBICACION": ciudad, estado, C.P. (3-col) + "Extraer del Chat" button
- Section: grupo + etiquetas (2-col card)

### Tab 2 — Ficha Curricular (value="academia")
Enhancements:
- Add "Nivel" dropdown to the "Registrar Nuevo Curso" form (Basico/Intermedio/Avanzado)
- Add attendance badge to each record in the history list
- Add attendance toggle button to each record entry
- Show nivel badge next to course name

### Tab 3 — Notas del Perfil (value="notas")
Add new section BEFORE the internal notes:
- Section "PERFIL DEL ALUMNO": 4 fields
  - dieta (textarea, 2 rows)
  - alimentacion (textarea, 2 rows)
  - alergias (textarea, 2 rows)
  - motivo_curso (textarea, 2 rows)
- Existing internal notes section stays below

## Changes Required

### 1. State (formData)
Add: `dieta`, `alimentacion`, `alergias`, `motivoCurso` to initial state and useEffect loader.

### 2. State (newCourse)
Add: `nivel` field.

### 3. useEffect (contact loader)
Load: `contact.dieta`, `contact.alimentacion`, `contact.alergias`, `contact.motivo_curso`.

### 4. handleSubmit
Save: `dieta`, `alimentacion`, `alergias`, `motivo_curso` to contacts table update.

### 5. Tab 2 UI
- Add nivel Select to "Registrar Nuevo Curso" form
- Add nivel to handleAddCourse record creation
- Show nivel badge in history list entries
- Add attendance toggle (checkbox or button) per entry
- Show attendance badge (green checkmark or red X)

### 6. Tab 3 UI
- Add "PERFIL DEL ALUMNO" section with 4 textareas before internal notes
- Wire onChange to formData setters

## Visual Style

Keep existing dark theme. No changes to color palette needed — already matches target:
- bg-slate-950 variants (bg-[#0f0f11], bg-[#0a0a0c], bg-[#161618])
- Indigo accents for academic tab
- Amber accents for notes tab
- Gold "GUARDAR EXPEDIENTE" button (currently indigo — change to amber/gold)

## Risks

- None significant. All columns exist. Pure frontend refactor.
- Backward compat: existing academic_record entries without `nivel`/`attendance` fields will render correctly (falsy defaults).
